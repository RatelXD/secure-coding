# Tiny Second-hand Shopping Platform 개발 전 과정 보고서

이 문서는 시큐어 코딩 과제로 진행하는 중고거래 플랫폼의 요구사항 분석, 시스템 설계, 구현, 테스트, 유지보수 과정을 한곳에서 확인할 수 있도록 정리한 보고서입니다. 최종 제출본은 이 Markdown 문서들을 바탕으로 사용자가 직접 PDF로 편집합니다.

## 현재 진행 상태

| 단계 | 상태 | 설명 |
|---|---|---|
| 요구사항 분석 | 1·2차 작성 완료 | 1차 구현 기준과 2차 검색·관리·모의 이체의 범위·정책·인수 조건을 확정했습니다. |
| 시스템 설계 | 1·2차 작성 완료 | 실제 1차 Django 단일 ASGI 구조와 구현 예정인 2차 데이터·권한·위협·트랜잭션 설계를 구분해 정리했습니다. |
| 구현 | 1차 범위 PASS·2차 구현 예정 | 사용자·상품·채팅·신고·가역 제재는 구현·검증했고, 검색·관리·모의 이체 제품 코드는 아직 없습니다. |
| 체크리스트와 테스트 | 1차 PASS·2차 구현 예정 | 1차 동작과 2차 상세 정책 구조를 포함한 자동 테스트 170개와 하위 사례 217개 및 컨테이너 복구 근거가 있습니다. 2차 제품 기능은 음성·경계·동시성·장애 테스트 계약만 작성했으며 실행하지 않았습니다. |
| 유지보수 | 1차 `MNT-01`~`MNT-04` PASS | 1차 구현 후 유지보수 점검에서 발견한 네 문제를 공개 issue, 최소 수정, 음성·회귀 테스트와 문서 갱신으로 처리했습니다. 2차 유지보수는 구현·검증 뒤 수행합니다. |
| 최종 제출 자료 | 작성 중 | 1차 본문·부록과 2차 설계는 정리했지만 2차 구현·검증·유지보수 근거가 생기기 전에는 최종 완료로 표시하지 않습니다. |

상태 표기는 다음 기준을 사용합니다.

- `구현 예정`: 설계만 확정되었고 코드가 아직 없습니다.
- `구현 중`: 코드 골격 또는 일부 구현이 있으나 종단 기능과 종합 검증이 끝나지 않았습니다.
- `미검증`: 확인 절차는 정의했지만 아직 실행하지 않았습니다.
- `확인 필요`: 근거가 불완전하거나 최신 코드에서 다시 확인해야 합니다.
- `PASS`: 실제 명령이나 수동 점검 결과가 남아 있는 경우에만 사용합니다.

2차 설계 완료는 구현 완료를 뜻하지 않습니다. 구현 전에 확정한 검증 항목은 [체크리스트와 테스트의 2차 개발 검증 계약](04-checklist-and-testing.md)에서 확인할 수 있습니다.
