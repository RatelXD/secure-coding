# 보안 기록 안내

과제 본문에서 다루는 제품 보안 약점, 개선 조치, 검증 상태는 [06. 보안 약점과 개선 계획](06-security-improvements.md)에 모았습니다. 이 기록에는 추적에 필요한 확인 결과만 남깁니다.

## 제품 보안 항목

| 식별자 | 항목 | 현재 상태 |
|---|---|---|
| `VULN-01` | 로그인 무차별 대입과 계정 정보 노출 | DB 권위 제한·일반 오류·병렬 경합·상태 정리 테스트 PASS |
| `VULN-02` | 타인 객체 접근과 대량 할당 | 본인·소유자·참여자 경계와 IDOR 음성 테스트 PASS |
| `VULN-03` | 위험한 상품 이미지 업로드 | 완전 디코딩·재인코딩과 우회 테스트 PASS |
| `VULN-04` | 채팅 권한 우회·XSS·중복 메시지 | 과거 잠금·Origin·휴면 수신 결함을 수정하고 집중 회귀 PASS |
| `VULN-05` | 신고 조작과 동시 제재 경합 | 원자적 서비스와 독립 동시성 테스트 PASS |
| `VULN-06` | 오류·로그의 민감정보 노출 | 일반 오류·비밀번호 로그 자동 테스트 PASS; 운영 오류 화면은 미검증 |
| `VULN-09` | 모의 이체 중복·부분 처리·잔액 불일치 | 서버 권위 이체·이중 분개·멱등성·대사 코드와 PR #43 CI PASS; 세부 장애·경합 재검증 필요 |
| `VULN-10` | 검색을 통한 비노출 정보 유출과 자원 고갈 | 공개 범위 선적용·NFC·allowlist·페이지 제한 코드와 PR #43 CI PASS; query budget 세부 재검증 필요 |
| `VULN-11` | 관리자 권한 상승과 감사 기록 훼손 | codename·grant·재인증·가역 제재·append-only 감사 코드와 PR #43 CI PASS; 전체 HTTP·감사 장애·경합 재검증 필요 |

환경 검증에서는 `.env` 없이 Compose를 시작해도 healthy 상태가 됐고, 정적 채팅 자산은 HTTP 200과 `text/javascript`로 제공됐습니다. 고정 버전으로 gitleaks 전체 이력을 검사했고, pytest 9.0.3 업그레이드 뒤 all-groups `pip-audit`도 모두 clean이었습니다. ngrok은 사용할 수 없어 검증하지 않았습니다.

PR #44에서는 WebSocket이 연결된 직후 반복적으로 끊기는 경우에도 재연결 카운터가 무한히 초기화되지 않도록 보정했습니다. 10초 안정 연결 뒤에만 카운터를 초기화하며, 실제 프록시·TLS 환경의 close code 전달은 미검증입니다.
