# 보안 기록 안내

과제 본문에서 다루는 제품 보안 약점, 개선 계획, 검증 상태는 [06. 보안 약점과 개선 계획](06-security-improvements.md)로 통합했습니다.

기존 문서에 있던 GitHub 리뷰 방식, 승인 표식, 배포 통제, 문서 렌더링 준비 기록은 제품의 보안 취약점과 직접 관계가 없어 제출 보고서 본문에서 제외했습니다. 해당 운영 과정에서 비밀값이나 개인정보를 공개 보고서에 복사하지 않는 원칙은 유지합니다.

## 제품 보안 항목

| 식별자 | 항목 | 현재 상태 |
|---|---|---|
| `VULN-01` | 로그인 무차별 대입과 계정 정보 노출 | DB 권위 제한·일반 오류 구현, 종합 재검증 필요 |
| `VULN-02` | 타인 객체 접근과 대량 할당 | 본인·소유자·참여자 경계 구현, 독립 음성 테스트 추가 |
| `VULN-03` | 위험한 상품 이미지 업로드 | 완전 디코딩·재인코딩과 우회 테스트 구현 |
| `VULN-04` | 채팅 권한 우회·XSS·중복 메시지 | 보안 경계 구현, PostgreSQL 행 잠금 결함 수정·재검증 필요 |
| `VULN-05` | 신고 조작과 동시 제재 경합 | 원자적 서비스 구현, 독립 동시성 테스트 PASS |
| `VULN-06` | 모의 이체의 중복·잔액 불일치 | 2차 범위, 미구현 |
| `VULN-07` | 오류·로그의 민감정보 노출 | 일반 오류·비밀번호 로그 음성 테스트 추가, 종합 재검증 필요 |

실제 취약점 수정 사례가 생기면 `기존 상태 → 개선 조치 → 음성 테스트 → 회귀 테스트 → 잔여 위험` 순서로 [06 문서](06-security-improvements.md)에 추가합니다.
